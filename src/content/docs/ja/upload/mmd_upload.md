> これで膨大な量の MMD モデルが使えるようになるよ ~~(まあ、めんどくさいんだけどね)~~

## 1. MMD モデルを Unity にインポート

[前の記事](/upload/mmd_to_fbx)で MMD モデルを FBX に変換する方法を説明したから、次は MMD モデルの全ファイルを Unity にインポートしよう。

## 2. MMD モデルの調整（セットアップ）

### 1）マテリアル

FBX ファイルをクリックして、右側の「Materials（マテリアル）」→「Location（場所）」→「Use External Materials（外部マテリアルを使用）」を選択。それから「Apply（適用）」をクリックして、しばらく待とう。

### 2）リグ（ボーン）

次に「Rig（リグ）」→「Animation Type（アニメーションタイプ）」→「Humanoid（ヒューマノイド）」を順に選んで、「Apply（適用）」をクリック。またしばらく待つよ。

それから「Configure…（設定）」をクリックしてボーンを確認しよう。

まずは T ポーズになってるか確認してね。もし違ってたら、右側の一番下までスクロールして「Pose（ポーズ）」→「Enforce T-Pose（T ポーズを強制）」をクリック。その後、ボーンに間違いがないかチェックしてね（大体「Head」にエラーが出やすいよ）。

確認が終わったら「Done（完了）」をクリック。

## 3. SDK 記述ファイル（Descriptor）の追加

### 1）視点（View）の調整

「VRC Avatar Descriptor」の一番上にあるのが「View（視点）」。これを調整するよ。「Edit（編集）」をクリックすると Unity 上で調整できるようになるから、だいたいおでこの前あたりに置いてね。終わったらスクリプトのところに戻って「Return（戻る）」をクリック。

### 2）「Viseme（リップシンク）」の追加

もし Blender で設定してないなら（[MMD モデルを FBX に変換するステップ 3](/upload/mmd_to_fbx/#%E4%B8%89%E8%BF%9B%E4%B8%80%E6%AD%A5%E5%A4%84%E7%90%86)をやってない場合）、この手順は無視して[ステップ 4](/upload/mmd_upload/#%E5%9B%9B%E6%B7%BB%E5%8A%A0%E5%8A%A8%E9%AA%A8%E5%92%8C%E7%A2%B0%E6%92%9E)に進むか、そのままアップロードしちゃって OK。

「VRC Avatar Descriptor」の 2 番目は「LipSync（リップシンク）」。「Mode（モード）」で「Viseme Blend Shape」を選んで、左側から「Body」を「Face Mesh」にドラッグすれば自動で認識されるよ。

### 3）「Eye Look（視線）」の追加

「VRC Avatar Descriptor」の 3 番目は「Eye Look」。「Enable（有効）」をクリックして、左右の目のボーンをそれぞれの場所にドラッグ。

下の「Eyelids（まぶた）」の「Eyelid Type」は「Blendshapes」を選んでから「Body」をドラッグすれば、これも自動認識されるよ。

## 4. ダイナミックボーンとコライダーの追加

{{% hint warning %}}
**注意**  
VRChat 公式から、Dynamic Bone はいずれ完全に削除されるってアナウンスされてるよ。新しい PhysBones を覚えるか追加するようにしてね。
{{% /hint %}}

ダイナミックボーンとコライダーの追加は何度も試行錯誤が必要で、結構面倒なんだ。でもこれを入れないと、ゲーム内で動いたときにモデルの服や髪が動かないんだよね。

### 1）ダイナミックボーン

「動かしたい」場所にダイナミックボーンを追加するよ。例えばスカートとか髪の毛とか。今回はスカートを例にするね。

スカートのボーンを見つけてクリックしたら、右側の「Add Component（コンポーネントを追加）」で「Dynamic Bone」を検索して追加。

次に、そのボーンを「Dynamic Bone」の「Root（ルート）」に入れて、4 つのパラメータを調整しよう。

Damping（減衰）：抵抗力
Elasticity（弾力）：跳ね返り

下の 2 つは制限範囲
Stiffness（剛性）：硬さ（下の数値より小さく設定）
Inert（不活性）：動きにくさ

以下の数値はあくまで参考だから、自分なりにベストな数値を試してみてね。

例：スカートのパラメータ 0.6、0.06、0.2、0.4

胸のパラメータ 0.08、0.06

髪のパラメータ 0.7、0.06、0.6、0.85

こちらのパラメータ参考：[VRchat 模型之 unity – 星空月零 – 博客园](https://www.cnblogs.com/raitorei/p/12008887.html)

    @莎莎 衣服 0.9  0.03  0.1  0

    @JIUJIU 特柔（突き抜けにくい設定） 0.7  0.06  0.9  0.9

    @Ryuu-San 耳と尻尾 0.7  0.07  0.666  0.52

こちらのパラメータ参考：[VRCHAT 模型上传常见问题汇总\_Ryuu-San 的博客-CSDN 博客](https://blog.csdn.net/TopOnelong/article/details/79002465)

### 2）コライダー（衝突判定）

ダイナミックボーンの設定が終わった後、「ゲームモード」でモデルを動かすと突き抜け（貫通）が確認できるはず。これはコライダーを追加することで解決できるよ。例えば足のコライダーとかね。

コライダーを追加したいボーンの下に新しい空のオブジェクトを作って（ボーンを右クリックして「Create Empty」）、そのオブジェクトをクリックして右側の「Add Component」から「Dynamic Bone Collider」を追加。サイズや高さを調整してね。

### 3）コライダーをダイナミックボーンに入れる

コライダーを干渉させたいダイナミックボーンがあるボーン（例えばスカート）をクリックして、さっき作ったオブジェクトを右側のダイナミックボーンコンポーネントの「Colliders（コライダー）」のリストにドラッグすれば OK（これで足のコライダーがスカートのボーンと干渉して、スカートがちゃんと持ち上がるようになるよ）。

これで「再生モード」にすれば効果が確認できるはず！
