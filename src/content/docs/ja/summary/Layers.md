> Unity のレイヤーの仕組み。モデルはどうやって変更されるのかな？

異なるレイヤーの状態は同時に発生できるけど、同じレイヤー内では一度に1つの状態しか実行できないんだ。

## レイヤー

`+` をクリックするとレイヤーを追加できるよ。レイヤーの隣にある歯車アイコンをクリックすると、小さなウィンドウが出てきて、そのレイヤーに対応するパラメータを設定できるんだ。

| 属性         | 説明                                                                                                                                            |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Weight(ウェイト) | この層の重み。0 はその層が効かないことを表し、1 はその層のアニメーションが完全に表現されることを表すよ。                                                    |
| Mask(マスク)   | この層で使用されるマスク。アニメーションの再生部位を制限できるんだ。例えば上半身だけアニメーションを再生したい場合、上半身に定義したマスクを使えばいい。設定すると、そのレイヤーの上に「M」の小さなアイコンが表示されるよ。 |
| Blending     | 他の層と混合するモード。                                                                                                                              |
| _Override_   | 上の層の、マスクに対応する部位を上書きするよ。                                                                                                                  |
| _Additive_   | 前のレイヤーのアニメーションの上に加算されるよ。                                                                                                                        |
| Sync         | 他の層のステートマシンを再利用するんだ。                                                                                                                            |
| IK Pass      | インバースキネマティクス（逆運動学）。                                                                                                                                      |

## 状態（ステート）

状態とはレイヤーの中にある「小さな四角」のことで、アニメーションをドラッグした時に自動的に生成される「モノ」のことだよ。

新しいレイヤーを作ると、3つの状態が見えるはず。

| 状態名  | 簡潔な説明                                           |
| :-------- | :--------------------------------------------- |
| Entry     | 入口。遷移（トランジション）に従ってデフォルトの状態に入るよ。                     |
| Any State | 任意の状態。条件さえ満たせば、どこからでも遷移先へジャンプできるんだ。 |
| Exit      | 出口。「Entry」に戻るよ。                            |

### 状態の追加

空白部分で右クリックして `Empty`（空の状態）を追加することもできるし、アニメーションファイルを `Animator` ウィンドウにドラッグして状態を追加することもできるよ。

もし `Project` ウィンドウでアニメーションファイルを選択しているなら、右クリックの `From Selected Clip` から状態を作ることもできるけど、直接ドラッグするのが一番手っ取り早いかな。

_最初に作った状態はデフォルトでオレンジ色になるんだ。これはデフォルトの状態を意味していて、`Animator` が開始時に再生するよ。状態の中にアニメーションがあれば、それも再生されるんだ。_

### 色

オレンジ色：デフォルトの状態。つまり、入った瞬間に実行されるアニメーション。

灰色：遷移（トランジション）によって条件を満たした時に実行されるアニメーション。

### 状態の設定

各状態にはアニメーションを1つ含めることができて、その状態にいる間、`Animator` コンポーネントがあるオブジェクトはそのアニメーションを再生するよ。状態を選択すると、次のようなオプションがあるんだ。

| 名称                   | 説明                                                                               |
| ---------------------- | ---------------------------------------------------------------------------------- |
| Motion                 | 状態に紐付けられたアニメーション（Animation）。                                                        |
| 速度(Speed)            | アニメーションの再生速度。                                                                     |
| 乗数(Multiplier)       | Speed * 指定したパラメータ（float）を実行時の再生速度にするよ。                                     |
| Motion Time            | 正規化された時間。指定したパラメータ（float、範囲は 0~1）をアニメーションの再生時間として使うんだ（これを使ってダイヤルスイッチとかが作れるよ）。    |
| ミラー(Mirror)           | 反転再生。「パラメータ(Parameter)」をオンにすると、指定したパラメータ（bool）を反転のスイッチにできるよ。                  |
| サイクルオフセット(Cycle Offset) | ループのオフセット量。ループアニメーションを同期させるのに使えて、正規化された時間（範囲は 0~1）を使うんだ。パラメータで制御もできるよ。 |
| Foot IK                | 人型（Humanoid）アニメーション専用。キャラの足に IK を使うかどうか。                                  |
| Write Default          | 次の状態に移る時に、この状態で加えた変更をリセットするよ（この状態で使っていないパラメータをデフォルト値に初期化するんだ）。         |

### Transition（遷移）

現在の状態の遷移を表示するよ。選択した後の内容はレイヤーで選択した時と同じだね。

| 名称 | 説明                               |
| ---- | ---------------------------------- |
| Solo | これにチェックを入れると、この遷移だけが有効になり、他は無効になるよ。 |
| Mute | これにチェックを入れると、この遷移が無効になるんだ。                 |

_もし `Solo` と `Mute` の両方にチェックを入れたら、`Mute` が優先されるよ。_

## 過渡（トランジション）

状態間の切り替え条件を表すよ。通常、ある状態から別の状態へ切り替えるための1つまたは複数の条件があるんだ。

状態を右クリックして `Make Transition` を選び、別の状態を選択すれば遷移が作れるよ。遷移の中で対応する条件を追加できるんだ。

| 名称 | 説明                               |
| ---- | ---------------------------------- |
| Solo | これにチェックを入れると、この遷移だけが有効になり、他は無効になるよ。 |
| Mute | これにチェックを入れると、この遷移が無効になるんだ。                 |

_もし `Solo` と `Mute` の両方にチェックを入れたら、`Mute` が優先されるよ。_

複数の遷移がある場合は、位置（上下）を調整して優先順位を決められるんだ（上が下より優先度が高いよ）。

| 名称          | 説明                                                         |
| ------------- | ------------------------------------------------------------ |
| Name Field    | 遷移に名前を付けられるよ。2つの状態の間に複数の遷移がある時に区別するのに便利。 |
| Has Exit Time | 終了時間を条件にするかどうか。                                         |

終了時間は特殊な遷移条件で、パラメータに依存せず、設定された終了タイミングを条件として状態を切り替えるんだ。

### Settings

遷移のいくつかのパラメータ設定だよ。

| 名称                              | 説明                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 終了時間(Exit Time)               | `Has Exit Time` にチェックが入っている場合に設定できるよ。アニメーションを終了する正規化された時間を設定するんだ（単位：前の状態のアニメーションの長さ。オブジェクトの起動アニメーションや空のアニメーションなら秒単位と考えてもいいよ）。<br />例えば 0.75 に設定すると、アニメーションが 75% まで再生された時に `true` になり、他の条件がなければそのまま次の状態に切り替わるんだ。<br />もし exit time が 1 未満なら、状態がループするたびにその位置で（アニメーションがループ設定かに関わらず、状態は常にループするよ）条件が true になる。例えば 1 回目の 75%、2 回目の 75%……といった感じだね。<br />もし exit time が 1 より大きいなら、条件の判定は 1 回だけ。例えば 3.5 なら、アニメーションが 3 回ループした後、4 回目の 50% に達した時に true になるよ。 |
| 固定期間(Fixed Duration)      | チェックを入れると、次の「遷移期間(Transition Duration)」が秒単位になるよ。<br />チェックを外すと、次の「遷移期間(Transition Duration)」が（前の状態の）パーセンテージになるんだ（1 は 100%）。                                                                                                                                                                                                                                                                                                                                                                                  |
| 遷移期間(Transition Duration) | 遷移にかかる時間。2 つの状態が切り替わる時、普通は一瞬で変わるんじゃなくて、スムーズに混ざりながら変わるんだ。このプロパティはそのスムーズに混ざる時間を設定するものだよ。                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| 遷移オフセット(Transition Offset)       | 状態が再生を開始する時の時間のオフセット。つまり、次の状態がどこから始まるか。値は次の状態のパーセンテージで、1 は 100% だよ。<br />例えば 0.5 に設定すると、次の状態に切り替わった時に 50% の位置から再生が始まるんだ。                                                                                                                                                                                                                                                                                                                                                                                                                     |

#### 割り込みソース(Interruption Source)

状態の遷移を途中で割り込ませることができるんだけど、かなり複雑だから、できるだけ使わない方がいいかもしれないね。

##### Current State（現在の状態）

デフォルトでは A から C への切り替えがトリガーされると、ステートマシンは C への切り替えを開始し、C に切り替わるまで変更できないんだ。でも、A->C の遷移の `割り込みソース(Interruption Source)` を `Current State` に変えると、A->C の遷移が A 上の**いくつか**の遷移によって割り込まれるようになるよ。

状態 A での遷移はこんな感じ：

![img](https://github.com/yexca/picx-images-hosting/raw/master/2022-VRChat/07-Layers/image.3te260424lu0.webp)

「いくつか」の理由：`Ordered Interruption` にチェックが入っている場合、A->C よりも優先順位が高いものだけが割り込めるんだ。

遷移の優先順位は上から下に向かって低くなる。つまり、優先順位は `A->B` > `A->C` > `A->D` になるよ。

- `Ordered Interruption` にチェックが入っている場合

  この時 A->B の切り替えがトリガーされると、A->C は中断されて A->B が実行されるよ。
  でも、この時 A->D がトリガーされても、A->C は中断されず、そのまま A->C が実行されるんだ。

- `Ordered Interruption` にチェックが入っていない場合

  この時は A->B も A->D も、トリガーされれば A->C を中断するよ。
  もし A->B と A->D が同時にトリガーされたら、優先順位が高い A->B がアクティブになるんだ。

##### Next State（次の状態）

もし A->C の遷移の `割り込みソース(Interruption Source)` を `Next State` に変えると、A->B や A->D は A->C を中断できなくなるんだ。

でも、もし A->C が開始された後に、B->D がアクティブになったら、A->C の遷移は中断されて D に切り替わるよ。

状態 B 上の遷移順序も影響するけど、この時は `Ordered Interruption` にチェックは入れられないんだ（A->B は状態 A のものだし、状態 B の順序には参加しないからね）。状態 B 上の遷移順序は、同時にトリガーされた時にどっちを使うかを決めるよ。例えば下の図のような遷移だね。

![](https://github.com/yexca/picx-images-hosting/raw/master/2022-VRChat/07-Layers/image.1xdggp8q1bq8.webp)

B->C と B->D が同時にトリガーされたら、優先的に B->C が実行されるよ。

##### Current State Then Next State（現在の状態、次に次の状態）

A->C の遷移の `割り込みソース(Interruption Source)` を `Current State Then Next State` にすると、状態 A と状態 B 両方の遷移が考慮されるようになるよ。

A->C への切り替え中に A->B, A->D, B->C, B->D が同時にアクティブになったとする（`Ordered Interruption` はオフね）。

この場合、まず状態 A の遷移を見て、A->B の優先順位が高いから A->B が実行されるよ。結果が出たから、次の状態 B のことはもう考えなくていいんだ。

##### Next State Then Current State（次の状態、次に現在の状態）

さっきのと逆の順番で判定するよ。

##### まとめ

| 名称                          | 説明                                                                                                                                      |
| ----------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| Ordered Interruption          | 現在の遷移よりも優先度が高い場合のみ割り込み可能。                                                                                                              |
| 無(None)                      | 割り込み不可。                                                                                                                                  |
| Current State                 | 前の状態から始まる遷移が、進行中の遷移を中断できる。                                                                                        |
| Next State                    | 次の状態から始まる遷移が、進行中の遷移を中断できる。                                                                                          |
| Current State Then Next State | どの状態の遷移ノードの優先権が高いかを決定する。まず前の状態の優先順位を見て、トリガーされなければ次の状態の優先順位を見ることで、異なる融合や中断効果を得るよ。 |
| Next State Then Current State | どの状態の遷移ノードの優先権が高いかを決定する。上と逆の順番。                                                                                   |

### Condition（条件）

遷移条件。1 つの遷移には 1 つの条件、複数の条件、あるいは条件なしの場合もあるよ。

- 条件がないけど `Has Exit Time` にチェックが入っている場合、`Exit Time` が終了条件として使われるよ。`Exit Time` に達すると次の状態に切り替わるんだ。

- 1 つまたは複数の条件がある場合、次の状態に切り替わるにはそれらの条件をすべて同時に満たす必要があるよ。

条件はパラメータを通じて設定できるんだ。具体的にはこんな感じ：

| パラメータ型 | 説明                                                                             |
| -------- | -------------------------------------------------------------------------------- |
| Int 型   | Greater（>）、Less（<）、Equals（==）、NotEqual（!=）で指定した数値を条件にするよ。 |
| Float 型 | Greater（>）、Less（<）で指定した数値を条件にするよ。                               |
| Bool 型  | 指定した値（true または false）を条件にするよ。                                        |
| Trigger  | 条件が true になった時に遷移をトリガーするよ。                                                       |

もし `Has Exit Time` にチェックが入っていて、さらに 1 つ以上の条件がある場合、遷移するには `Exit Time` に到達し、かつ全ての条件が true である必要があるんだ。

> 遷移には少なくとも 1 つの条件（Has Exit Time も 1 つの条件として数えるよ）が必要だよ。そうじゃないと、その遷移は無視されて次の状態には行けないんだ。

注：_Trigger は VRChat ではあんまり使われないみたい。少なくともボクは使い道を知らないな。_
