> Unity トレイルレンダラー

トレイルレンダラー (Trail Renderer) コンポーネントは、移動するゲームオブジェクトの後ろにポリゴンの軌跡をレンダリングするよ。このコンポーネントを使うと、移動するオブジェクトの躍動感を強調したり、その経路や位置を際立たせたりできるんだ。ミサイルの後ろに伸びる軌跡は飛行ルートを視覚的に分かりやすくしてくれるし、飛行機の翼の端から出る飛行機雲は、現実世界でのトレイル効果のいい例だね。

## 作成方法

ヒエラルキー（Hierarchy）で右クリックして、`Effects（エフェクト） - Trail（トレイル）` を選択するよ。

## トレイルの設定

| 属性                                     | 機能                                                                                                                 |
| ---------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **Width（幅）**                          | 軌跡の長さに沿った幅を制御するために、幅の値とカーブの値を定義する。ダブルクリックすると新しいキー（頂点）を作成できるよ。                                                       |
| **Time（時間）**                           | 軌跡上の各ポイントの生存期間（秒単位）を定義するよ。                                                                             |
| **Min Vertex Distance（最小頂点間距離）**    | 軌跡の2点間の最小距離（ワールド単位）。                                                                             |
| **AutoDestruct（自動破棄）**               | このプロパティを有効にすると、`Trail Renderer` コンポーネントがアタッチされているゲームオブジェクトが静止してから `Time` 秒後に、そのゲームオブジェクトが自動的に破棄されるよ。                                     |
| **Emitting（エミッション）**                   | このプロパティを有効にすると、Unityは軌跡に新しいポイントを追加するよ。無効にすると追加されなくなる。軌跡の生成を一時停止したり再開したりするのに使ってね。 |
| **Color（色）**                          | 軌跡の長さに沿った色を制御するグラデーションを定義するよ。                                                                                 |
| **Corner Vertices（コーナーの頂点数）**              | 軌跡の角を描画するときに、どれだけの追加の頂点を使用するかを指定する。この値を増やすと、軌跡の角がより丸くなるよ。                                         |
| **End Cap Vertices（端の頂点数）**           | 軌跡の端にキャップ（丸み）を作成するために、どれだけの追加の頂点を使用するかを指定する。この値を増やすと、軌跡の端がより丸くなるよ。                                         |
| **Alignment（整列）**                      | 軌跡が向く方向を設定するよ。                                                                                                   |
| _View（ビュー）_                             | 軌跡がカメラを向くよ。                                                                                                       |
| _TransformZ（トランスフォーム Z）_                     | 軌跡がトランスフォーム（Transform）コンポーネントのZ軸方向を向くよ。                                                                                            |
| **Texture Mode（テクスチャモード）**               | テクスチャを軌跡にどのように適用するかを制御するよ。                                                                                             |
| _Stretch（ストレッチ）_                          | テクスチャを軌跡の長さ全体に1回だけマッピングするよ。                                                                                         |
| _Tile（タイル）_                             | 軌跡の長さ（ワールド単位）に基づいて、軌跡に沿ってテクスチャを繰り返す。タイリング率を設定するには、`Material.SetTextureScale()` を使ってね。                        |
| _DistributePerSegment（セグメントごとに分配）_         | テクスチャを軌跡全体に1回マッピングする（すべての頂点が均等に分布していると仮定するよ）。                                                                 |
| _RepeatPerSegment（セグメントごとに繰り返す）_             | テクスチャを軌跡のセグメントごとに繰り返す。タイリング率を調整するには、`Material.SetTextureScale()` を使ってね。                              |
| **Generate Lighting Data（ライトデータの生成）** | このプロパティを有効にすると、Unityは軌跡のジオメトリを構築するときに法線と接線を含めるようになるよ。これで、シーンのライティングを反映するマテリアルが使えるようになるんだ。               |
| **Shadow Bias（シャドウバイアス）**                | シャドウのアーティファクト（にじみや崩れ）を防ぐために、ライトの方向に沿ったシャドウの移動量を設定するよ。                                                                           |

## Materials（マテリアル）

`Trail Renderer` インスペクターの `Materials` セクションには、現在使われているマテリアルの一覧が表示されるよ。複数のマテリアルを適用すると、それぞれのマテリアルで1回ずつ、計複数回レンダリングされるんだ。

| 属性              | 功能                                                                                                 |
| :---------------- | :--------------------------------------------------------------------------------------------------- |
| **Size（サイズ）**    | `Trail Renderer` で使用するマテリアルの数を指定するよ。リストのサイズを小さくすると、リストの末尾から要素が削除されるんだ。 |
| **Element（要素）** | `Trail Renderer` に設定されているマテリアルのリスト（番号順）。                                                |

## Lighting（ライティング）

`Lighting` セクションには、このトレイルレンダラーが Unity のライトとどのように相互作用するかを設定するプロパティがあるよ。

| 属性                          | 功能                                                                                          |
| :---------------------------- | :-------------------------------------------------------------------------------------------- |
| **Cast Shadows（シャドウをキャスト）**    | 適切なライトが軌跡に当たったときに、その軌跡がシャドウをキャスト（投影）するかどうか、またどのように投影するかを指定するよ。                                    |
| _Off（オフ）_                   | 軌跡はシャドウをキャストしないよ。                                                                            |
| _On（オン）_                    | シャドウをキャストするライトが当たったときに、軌跡がシャドウをキャストするよ。                                                  |
| _Two Sided（両面）_             | 軌跡の両側からシャドウをキャストする。ただし、`Enlighten` や `Progressive Lightmapper（プログレッシブライトマッパー）` は両面シャドウをサポートしていないから注意してね。 |
| _Shadows Only（シャドウのみ）_        | 軌跡のシャドウだけが表示されて、軌跡自体は非表示になるよ。                                                              |
| **Receive Shadows（シャドウを受ける）** | このオプションを有効にすると、軌跡の上に他の影が落ちるようになるよ。このオプションは、`Progressive Lightmapper` を使用している場合のみサポートされているんだ。          |

## Probes（プローブ）

`Probes` セクションには、[ライトプローブ（Light Probes）](https://docs.unity3d.com/ja/2019.4/Manual/LightProbes.html) や [リフレクションプローブ（Reflection Probes）](https://docs.unity3d.com/ja/2019.4/Manual/class-ReflectionProbe.html) に関するプロパティが含まれているよ。

| 属性                              | 功能                                                                                                                                           |
| :-------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------- |
| **Light Probes（ライトプローブ）**      | このレンダラーがライトプローブシステムからどのようにライトを受け取るかを設定するよ。                                                                                                       |
| _Off（オフ）_                       | レンダーは補間されたライトプローブを一切使用しないよ。                                                                                                                   |
| _Blend Probes（プローブをブレンド）_        | レンダーは1つの補間されたライトプローブを使用するよ。                                                                                                                     |
| _Use Proxy Volume（プロキシボリュームを使用）_    | レンダーは、補間されたライトプローブの3Dグリッドを使用するよ。                                                                                                               |
| _Custom Provided（カスタム提供）_     | レンダーは [MaterialPropertyBlock](https://docs.unity3d.com/ja/2019.4/ScriptReference/MaterialPropertyBlock.html) からライトプローブシェーダーのUniform値を取得するよ。 |
| Proxy Volume Override（プロキシボリュームのオーバーライド） | `Light Probe Proxy Volume` コンポーネントを持つ別のゲームオブジェクトへの参照を設定するよ。このプロパティは、`Light Probes` が `Use Proxy Volume` に設定されている場合のみ表示されるんだ。                     |
| **Reflection Probes（リフレクションプローブ）** | レンダーラーがリフレクションプローブシステムからどのように反射を受け取るかを設定するよ。                                                                                                         |
| _Off_                             | リフレクションプローブを無効にするよ。Unityは反射にスカイボックス（Skybox）を使用するんだ。                                                                                                           |
| _Blend Probes_                    | リフレクションプローブを有効にするよ。ブレンドはリフレクションプローブ間でのみ行われるんだ。屋内環境で、キャラクターが異なるライト設定のエリア間を移動するときに便利だよ。                             |
| _Blend Probes and Skybox_         | リフレクションプローブを有効にするよ。ブレンドはリフレクションプローブ間、またはリフレクションプローブとデフォルトの反射との間で行われるんだ。屋外の環境で役に立つよ。                                                             |
| _Simple_                          | リフレクションプローブを有効にするけど、2つの重なるプローブボリュームがあっても、リフレクションプローブ間のブレンドは行われないよ。                                                                             |
| **Anchor Override（アンカーのオーバーライド）**     | ライトプローブやリフレクションプローブシステムを使用する際に、Unityが補間位置を決定するために使用するトランスフォーム（Transform）コンポーネントを設定するよ。デフォルトでは、この位置はレンダラーのジオメトリのバウンディングボックスの中心になるんだ。                      |

## Additional Settings（追加の設定）

| 属性                               | 功能                                                                                                                                                                                                                                                                                                |
| :--------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Motion Vectors（モーションベクトル）**       | このレンダラーのフレーム間のピクセル単位の画面空間の動きを追跡するために、モーションベクトルを使用するかどうかを設定するよ。この情報を使うことで、モーションブラーなどのポストプロセス効果を適用できるようになるんだ。ただし、すべてのプラットフォームがモーションベクトルをサポートしているわけではないから注意してね。                                                                                                                                              |
| _Camera Motion Only（カメラの動きのみ）_ | カメラの移動のみを使って動きを追跡するよ。                                                                                                                                                                                                                                                                          |
| _Per Object Motion（オブジェクトごとの動き）_    | このレンダラーの動きを追跡するために、特定のパスを使用するよ。                                                                                                                                                                                                                                                                    |
| _Force No Motion（動きなしを強制）_      | 動きを追跡しないよ。                                                                                                                                                                                                                                                                                          |
| **Dynamic Occlusion（動的オクルージョン）**    | `Dynamic Occlusion` を有効にすると、カメラの視野内でこのレンダラーが静的オクルーダー（Static Occluder）に遮られたときに、Unityはこのレンダラーのカリング（描画スキップ）を行うよ。<br />無効にすると、静的オクルーダーに遮られていてもカリングされなくなるんだ。`Dynamic Occlusion` を無効にすると、壁の後ろにキャラクターの輪郭線を描画するようなエフェクトが実現できるよ。 |
| **Sorting Layer（ソーティングレイヤー）**        | このレンダラーのソーティングレイヤー（Tags and Layers）の名前だよ。                                                                                                                                                                                                                                                           |
| **Order in Layer（レイヤー内の順序）**       | このレンダラーのソーティングレイヤー（Tags and Layers）内での描画順序だよ。                                                                                                                                                                                                                                                         |
