> Unityのクロス（布）システム

`Cloth（クロス）`コンポーネントは、`Skinned Mesh Renderer（skinned メッシュレンダラー）`と連動して、物理ベースの布シミュレーションを提供するものだよ。このコンポーネントはキャラクターの衣装用に設計されていて、スキンメッシュにのみ有効なんだ。もしスキンメッシュじゃない通常メッシュに`Cloth`コンポーネントを追加すると、Unityは自動的にその通常メッシュを削除して、スキンメッシュを追加するよ。

## メッシュとウェイト（骨骼权重）の処理

クロスとDynamic Bone（動骨）は別々のシステムだから、共存させるには適切な処理が必要なんだ。

大体以下のような要件があるよ。

- シングルレイヤー（単層）メッシュ

  服が単層（1枚のメッシュ）であることを確認しよう。

- メッシュの結合（縫合）

  服がバラバラにならず、一体になっていることを確認しよう。

- 干渉の分離

  服についている小さなアクセサリーとか小物は、クロスシミュレーションの邪魔になることがあるんだ。

- 四角形メッシュ

  服のメッシュが三角形ばかりだと、あまり綺麗なシミュレーション結果にならないことがあるよ。

### メッシュを結合（縫合）する方法は？

Blenderで重複する頂点を削除してマージするんだ。トップメニューの「メッシュ」＞「クリーンアップ」＞「距離でマージ」を選んで、左下の「距離でマージ」パネルで「結合距離」の数値を大きくすると、隣り合う2つの点が合体するよ。ショートカットキーは `M` だよ。

## クロスの追加

「Inspector」ウィンドウで「Add Component」ボタンをクリックして、「Physics」＞「Cloth」を選択しよう。

## Edit Cloth Constraints（制約の編集）

クロスの特定の頂点に対して制約を適用することで、それらの頂点の動きの自由度を増やしたり減らしたりできるよ。

### Cloth Constraints

| 属性                | オプション             | 機能                                                                                                                         |
| :------------------ | :--------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **Visualization**   |                        | 表示する制約タイプとパーティクルを選択するよ。                                                                                                 |
|                     | _Max Distance_         | クロスパーティクルの Max Distance（最大距離）の値だけを表示するよ。                                                                                           |
|                     | _Surface Penetration_  | クロスパーティクルの Surface Penetration（表面貫通）の値だけを表示するよ。                                                                                    |
|                     | _Manipulate Backfaces_ | このオプションを有効にすると、クロスの裏側に隠れているパーティクルを視覚化して操作できるようになるよ。                                                             |
| [ カラーマップ ]            |                        | クロス全体に適用されている現在の最小値和最大値に基づいて、選択した制約タイプに対応するパーティクルの色と制約値の関係を示すよ。黒は常に制約がないことを意味しているよ。 |
| **Constraint Size** |                        | クロスパーティクルを表す球体の表示サイズ。編集しやすいように調整できるけど、制約そのものには何の影響もないよ。                         |

### 制約タイプと説明

| 制約タイプ            | 説明                             |
| ------------------- | -------------------------------- |
| Max Distance        | クロスが頂点位置から移動できる最大距離 |
| Surface Penetration | クロスパーティクルがメッシュを貫通できる深さ         |

「Visualization」で選択したタイプに基づいて、パーティクルの色はクロス内での制約の相対値を表すよ。

### 制約の編集モード

クロスの制約値を編集するのに、3つの異なるモードが使えるよ。

#### 1）Select

> 選択したパーティクルグループに固定の制約値を適用するよ

マウスでドラッグして範囲選択したあと、適用したい制約タイプを有効にして値を設定するだけでOKだよ。

#### 2）Paint

> ブラシでクロスパーティクルを塗ることで、固定の制約値を適用するよ

必要に応じて「Brush Radius（ブラシの半径）」を調整して、適用したい制約タイプを有効にして値を設定したら、ブラシでパーティクルの上をなぞって塗っていこう。

#### 3）Gradient

> 事前に選択したパーティクルグループに対して、左から右へ線形グラデーションの制約値を適用するよ

**あらかじめ「Scene（シーン）」ビューで「2D」ビューを有効にしておく必要があるから注意してね。**

マウスで範囲選択したあと、範囲を設定し、適用したい制約タイプを選択するよ。

## Edit Cloth Self/Inter-Collision（自己衝突と相互衝突の編集）

- 自己衝突（Self-Collision）：クロスが自分自身を突き抜けるのを防ぐよ。
- 相互衝突（Inter-Collision）：クロスパーティクル同士がぶつかり合うのを許可するよ。

### Cloth Self-Collision and Inter-Collision

| 属性                     | オプション        | 機能                                                             |
| :----------------------- | :---------------- | ---------------------------------------------------------------- |
| **Visualization**        |                   | 表示する制約タイプとパーティクルを選択するよ。                                     |
|                          | _Self-Collision_  | 自己衝突                                                           |
|                          | _Inter-Collision_ | 相互衝突                                                           |
| **Manipulate Backfaces** |                   | このオプションを有効にすると、クロスの裏側に隠れているパーティクルを視覚化して操作できるようになるよ。 |

### 色

| 色 | 説明           |
| ---- | -------------- |
| 黒 | 未使用のパーティクル   |
| 青 | 現在選択されているパーティクル   |
| 緑 | 衝突が適用されているパーティクル |

### 適用方法

「Select」を選んで、マウスで範囲選択したあと「Self-Collision and Inter-Collision」にチェックを入れればOKだよ。

#### Self-Collision

> 自己衝突と相互衝突を適用すると、インスペクター（Inspector）にこのオプションが表示されるようになるよ。

このとき、シーン上のパーティクルが消えちゃうことがあるかもしれないけど、インスペクターの「Self-Collision Distance」の値を大きくすれば見えるようになるよ。

| 属性          | 機能                                                                                                                                                                                |
| :------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Distance**  | 各パーティクルを包む球体の直径だよ。Unityはシミュレーション中にこれらの球体が重ならないように制御するんだ。**Distance** の値は、構成されている2つのパーティクル間の最小距離よりも小さくする必要があるよ。もしこの値が大きすぎると、自己衝突が一部の距離制約に違反して、ガタガタと小刻みに揺れる原因（ジッター）になっちゃう。 |
| **Stiffness** | パーティクル同士の反発力の強さだよ。この値はクロスソルバーによって計算されて、パーティクル同士を十分に引き離すのに役立つよ。                                                                             |

### Paint（塗る）と Erase（消去）

「Brush Radius」を調整したら、シーン上をドラッグして、衝突判定に使うパーティクルを追加したり削除したりできるよ。

### 相互衝突

パーティクルの指定方法は自己衝突と同じだけど、設定を行うには「Edit」＞「Project Settings」＞「Physics」で「Cloth Inter-Collision」にチェックを入れる必要があるよ。そのあとで「Distance」と「Stiffness」（機能は自己衝突と同じだよ）を設定してね。

### 注意点

自己衝突と相互衝突は、シミュレーション全体の計算負荷がかなり大きくなる可能性があるんだ。衝突距離を小さくしたり、自己衝突インデックスを使って衝突するパーティクル数を減らすことを検討してみてね。

自己衝突は三角形（ポリゴン）ではなく頂点を使用するから、三角形のサイズがクロスの厚みよりもはるかに大きいメッシュの場合、衝突判定が完璧に機能することは期待できないよ。

## コリジョン（衝突判定）

クロスは「Capsule Collider（カプセルコライダー）」または「Sphere Collider（球体コライダー）」としか相互作用できないんだ。そのうち2つの「Sphere Collider」を連結させて、円錐形のコライダーにすることもできるよ。

コライダーの「Is Trigger」にチェックを入れるのを忘れないでね。そうしないと、ゲーム内で移動したりジャンプしたりしたときに、自分で自分の足を踏んでいるようなおかしな挙動になって、ぐるぐる回っちゃったりするよ。

### カプセルコライダー（Capsule Collider）

カプセルコライダーは、2つの半球と1つの円柱が合体してできた形状のコライダーだよ。

#### 追加方法

該当するボーンの下に空のゲームオブジェクトを作成して、「Inspector」ウィンドウで「Add Component」ボタンをクリックし、「Physics」＞「Capsule Collider」を選択しよう。

#### 属性

| 属性              | 機能                                                       |
| :---------------- | :--------------------------------------------------------- |
| **Edit Collider** | 選択すると、「Scene」ビュー上で視覚的にサイズを編集できるよ。                           |
| **Is Trigger**    | このプロパティを有効にすると、トリガーイベント用として使われ、物理エンジンによる物理的な衝突は無視されるよ。 |
| **Material**      | 物理マテリアル（Physic Material）を割り当てて、他のオブジェクトとの相互作用のしかたを決めるよ。           |
| **Center**        | オブジェクトのローカル空間におけるコライダーの中心位置だよ。                               |
| **Radius**        | コライダーのローカル幅の半径だよ。                                     |
| **Height**        | コライダーの全高（全体の高さ）だよ。                                             |
| **Direction**     | カプセルがオブジェクトのローカル空間でどの軸方向に伸びるか。                       |

#### クロスへの追加

コライダーが設定されているオブジェクトを、そのまま「Cloth」コンポーネントの「Capsule Colliders」スロットにドラッグ＆ドロップすればOKだよ。

### 球体コライダー（Sphere Collider）と円錐

球体コライダーは、基本的な球体の形をしたシンプルな3Dコライダーだよ。

#### 追加方法

該当するボーンの下に空のゲームオブジェクトを作成して、「Inspector」ウィンドウで「Add Component」ボタンをクリックし、「Physics」＞「Sphere Collider」を選択しよう。

#### 属性

| 属性              | 功能                                                       |
| :---------------- | :--------------------------------------------------------- |
| **Edit Collider** | 選択すると、「Scene」ビュー上で視覚的にサイズを編集できるよ。                           |
| **Is Trigger**    | このプロパティを有効にすると、トリガーイベント用として使われ、物理エンジンによる物理的な衝突は無視されるよ。 |
| **Material**      | 物理マテリアルを割り当てて、他のオブジェクトとの相互作用のしかたを決めるよ。           |
| **Center**        | オブジェクトのローカル空間におけるコライダーの中心位置だよ。                               |
| **Radius**        | コライダーの半径の大きさだよ。                                           |

#### クロスへの追加

「Cloth」コンポーネントの「Sphere Colliders」の「Size」を0以外の値に調整すると、各要素（Element）に最大2つの球体コライダーを登録できるようになるよ。

1つだけ登録した場合は、通常の球体コライダーとして機能するよ。

2つ登録した場合は、それら2つの球体コライダーが自動的につながって、円錐形のコライダーとして機能するんだ。
